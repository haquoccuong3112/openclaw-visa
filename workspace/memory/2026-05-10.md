## Pipeline rules + bottleneck analysis (chốt với Cường 2026-05-10 tối)
- Chốt rule OCR pipeline:
  - Default: `google/gemini-2.0-flash-lite-001`
  - Fallback 1: `google/gemini-2.5-flash-lite`
  - Fallback 2 (giấy khó): `google/gemini-2.5-flash`
  - Fallback OCR chuẩn: Google Document AI (`scan-ho-so-ocr`, region `us`, processor `3183188b763e1843`)
- End-to-end run của bộ Hoàng Thị Mơ (18 file): khoảng 13 phút tổng (gồm thử/sai Document AI). Riêng nhánh Gemini 2.5 Flash chỉ ~2 phút 30 giây cho 18 file.
- Bottleneck thật sự không phải OCR model nữa, mà là orchestration:
  1. Drive API upload + check/list folder lặp nhiều request mỗi file (cần cache folder ID, batch).
  2. OCR đang chạy tuần tự từng file (nên giới hạn parallel 3–5 file/case).
  3. Sheet append nếu nhiều worker chạy song song dễ race/duplicate, cần lock theo Case ID và batch append 1 lần/case.
- Ước tính nếu 5 nhóm cùng gửi bộ ~18 file: kiểu hiện tại ~15–25 phút; sau tối ưu (queue + cache + parallel có giới hạn) khoảng ~4–7 phút cho 5 case song song, mỗi case 1–2 phút.
